Contexte du stage

Ce stage de première année de BTS SIO m'a permis de découvrir le développement mobile avec React Native, un framework JavaScript permettant de créer des applications natives pour iOS et Android à partir d'une seule base de code.

Pourquoi React Native ?

React Native est aujourd'hui l'un des frameworks mobiles les plus populaires, utilisé par des géants comme Facebook, Instagram, Uber, Airbnb, et bien d'autres. Ses principaux avantages :

  • Code partagé entre iOS et Android : un seul codebase pour deux plateformes
  • Performances natives : contrairement aux solutions hybrides, React Native compile vers du code natif
  • Écosystème riche : accès à npm et aux milliers de bibliothèques JavaScript
  • Hot Reload : voir les modifications en temps réel sans recompiler l'application
  • Communauté active : documentation fournie, tutoriels, support communautaire

Semaine 4 : Projet principal - Application de recherche de films

Le projet phare du stage consistait à développer une application mobile de recherche de films en utilisant l'API The Movie Database (TMDB).

Projet : MovieSearch App 🎬

Présentation du projet

MovieSearch est une application mobile permettant de rechercher des films, consulter leurs détails (résumé, note, affiche, date de sortie), et découvrir les tendances du moment. L'application communique avec l'API TMDB pour récupérer des données en temps réel.

Fonctionnalités développées

🔍 Recherche de films

  • Barre de recherche avec suggestions en temps réel
  • Debounce pour limiter les appels API
  • Affichage des résultats en grille avec affiches
  • Gestion des états vides (aucun résultat, en attente de saisie)
  • Pull-to-refresh pour actualiser les résultats

📄 Détails du film

  • Affichage complet : affiche, titre, résumé, date de sortie
  • Note moyenne et nombre de votes
  • Genres du film (Action, Comédie, Drame, etc.)
  • Durée du film en minutes
  • Liste des acteurs principaux
  • Films similaires recommandés

🔥 Films populaires

  • Section "Tendances du moment"
  • Films les mieux notés
  • Prochaines sorties au cinéma
  • Carrousel horizontal scrollable
  • Actualisation quotidienne automatique

⭐ Favoris

  • Ajout/suppression de films en favoris
  • Persistance des données avec AsyncStorage
  • Écran dédié aux favoris
  • Icône cœur animée
  • Badge indiquant le nombre de favoris

Architecture et organisation du code

Le projet a été structuré selon une architecture claire et maintenable :

  • /screens : écrans de l'application (HomeScreen, SearchScreen, DetailScreen, FavoritesScreen)
  • /components : composants réutilisables (MovieCard, SearchBar, LoadingSpinner, RatingStars)
  • /services : logique d'appels API et gestion des données (tmdbApi.js)
  • /navigation : configuration de React Navigation (Stack, Tab, Drawer navigators)
  • /utils : fonctions utilitaires (formatDate, truncateText, debounce)
  • /assets : images, icônes, fonts

Extraits de code significatifs

Exemple : Service API pour récupérer les films populaires 
import axios from 'axios';

const API_KEY = 'votre_clé_api_tmdb';
const BASE_URL = 'https://api.themoviedb.org/3';

export const getPopularMovies = async () => {
  try {
    const response = await axios.get(
      `${BASE_URL}/movie/popular?api_key=${API_KEY}&language=fr-FR`
    );
    return response.data.results;
  } catch (error) {
    console.error('Erreur lors de la récupération des films:', error);
    throw error;
  }
};

Défis techniques et solutions

Défi 1 : Gestion de la performance avec de longues listes
L'affichage de centaines de films ralentissait considérablement l'application.

Solution : Utilisation de FlatList au lieu de ScrollView pour bénéficier du lazy loading et de la virtualisation. Implémentation de la pagination avec chargement progressif des résultats ("infinite scroll"). Optimisation des images avec des dimensions appropriées et mise en cache.

Défi 2 : Appels API excessifs lors de la saisie
Chaque frappe déclenchait un appel API, saturant rapidement les limites de l'API gratuite.

Solution : Mise en place d'un mécanisme de debounce : l'appel API n'est effectué que 500ms après la dernière frappe de l'utilisateur. Cela réduit drastiquement le nombre d'appels tout en gardant une expérience fluide.

Défi 3 : Persistance des favoris
Besoin de sauvegarder les films favoris localement pour qu'ils persistent entre les sessions.

Solution : Utilisation d'AsyncStorage, l'équivalent de localStorage pour React Native. Création d'un hook personnalisé useFavorites pour gérer l'ajout, la suppression et la récupération des favoris de manière centralisée.

Défi 4 : Navigation entre écrans avec passage de données
Difficulté à transmettre les données d'un film depuis la liste vers l'écran de détails.

Solution : Maîtrise de React Navigation et de son système de paramètres de route. Passage des données du film via navigation.navigate('Details', { movie }) et récupération avec route.params.movie dans l'écran de destination.
